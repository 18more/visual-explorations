SVG sizing rules

1️⃣ if the object (your svg) has explicit width, height, and ratio, use them
2️⃣ if the object has only ratio, use auto for both width and height while maintaining the ratio
3️⃣ if none of these dimensions is available, use width 300px and height 150px when the viewport is larger than 300px, and auto for width and height with a ratio of 2:1 when the viewport is smaller than 300px

SVG element with no styling, and only a viewBox="0 0 24 24" attribute

There are no width or height, but there is a ratio. Use a width of auto, which in this case means fill the width of the enclosing container. Then using the calculated width and the ratio inferred from viewBox, calculate the height.

SVG element with no styling, and no attributes

Width and height are not specified, and there’s no viewBox attribute from which the ratio could be inferred. Rule 3️⃣ will be applied:

3️⃣ if none of these dimensions is available, use width 300px and height 150px when the viewport is larger than 300px, and auto for width and height with a ratio of 2:1 when the viewport is smaller than 300px

The path inside the SVG element is defined using unitless numbers, which seem to map onto pixels. So if the SVG has a width of 300px and a height of 150px. And you have a rect of width "100", then that’s going to be pixels.
