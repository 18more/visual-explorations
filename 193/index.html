193 - content: on elements and pseudo-elements

There seems to be a difference between how content-replacement is treated for elements and pseudo-elements. What seems to be happening is that the single url still means content-list for pseudo-elements, and it does mean content-replacement only for elements.

Letâ€™s translate this into an analogous markup for pseudo-elements. The div is the pseudo-element, and the img is the inline anonymous replaced element representing the child box of the pseudo-element:

      <div>
        <img src="" alt="">
      </div>

For elements, it really becomes equivalent to using an img element. Note that we can use properties like object-fit on it.

      <img src="" alt="">

A section element originating a display: block pseudo-element with content: content-replacement. 

        .one::before {
          content: url('https://picsum.photos/id/237/200/300');
          display: block;
          width: 100%;
          height: 300px;
          object-fit: cover;
          background-color: wheat;
        }

A section element with content: content-replacement. 

      .two {
        content: url('https://picsum.photos/id/237/200/300');
        display: block;
        width: 100%;
        height: 300px;
        object-fit: cover;
        background-color: wheat;
      }

An actual image element. 

      img {
        display: block;
        width: 100%;
        height: 300px;
        object-fit: cover;
        background-color: wheat;
      }
