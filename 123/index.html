Floats in a nutshell
Tuesday, 13 July 2021 

- floated elements don’t contribute to the height of their auto-sized container, unless that container is a block formatting context root
- the root element (html) is an example of a block formatting context root
- that’s why you may have a body element with some text and a floated image, and the body element will have a height that only includes the text, but the root element will include the floated descendant, too
- these things establish a new block formatting context:
-- overflow other than visible
-- display: inline-block
-- position: absolute
